# Macaw
### The AI Coding Agent Usage Tracker

Macaw is a local web application that ingests session logs from AI coding agents, stores usage data in SQLite, and displays token consumption summaries via a browser-based dashboard.

## Supported Agents

| Agent | Log Location | Token Field |
|---|---|---|
| Claude Code | `~/.claude/projects/**/*.jsonl` | `usage` on `assistant` message entries |
| Codex CLI | `~/.codex/sessions/YYYY/MM/DD/*.jsonl` | `token_count` event_msg entries |
| pi.dev | `~/.pi/agent/sessions/**/*.jsonl` | `usage` on `message` entries with `role: "assistant"` |

All log formats are JSONL. Each session is a single file. Token data is embedded in the logs — no hooks or instrumentation required.

## Stack

- **Backend:** NestJS (TypeScript)
- **Database:** SQLite via `better-sqlite3` (synchronous API, no ORM overhead)
- **Frontend:** Plain HTML + Chart.js served as NestJS static files — no build step
- **Scheduler:** `@nestjs/schedule` for automatic ingestion every 5 minutes and on startup

## Database Schema

**`sessions`** — one row per JSONL file:

| Column | Type | Notes |
|---|---|---|
| `session_id` | TEXT PK | UUID from filename |
| `source` | TEXT | `claude`, `codex`, or `pi` |
| `project_path` | TEXT | Derived from parent directory name |
| `model` | TEXT | |
| `first_timestamp` | TEXT | |
| `last_timestamp` | TEXT | |
| `input_tokens` | INTEGER | |
| `output_tokens` | INTEGER | |
| `cache_creation_tokens` | INTEGER | Claude: `cache_creation_input_tokens` |
| `cache_read_tokens` | INTEGER | Claude: `cache_read_input_tokens` |
| `reasoning_tokens` | INTEGER | Codex: `reasoning_output_tokens` |
| `last_line` | INTEGER | Incremental ingestion cursor |

**`messages`** — one row per assistant turn with usage:

| Column | Type |
|---|---|
| `id` | INTEGER PK |
| `session_id` | TEXT FK |
| `source` | TEXT |
| `timestamp` | TEXT |
| `model` | TEXT |
| `input_tokens` | INTEGER |
| `output_tokens` | INTEGER |
| `cache_creation_tokens` | INTEGER |
| `cache_read_tokens` | INTEGER |
| `reasoning_tokens` | INTEGER |

## API Surface

```
POST /api/ingest              Trigger ingestion manually
GET  /api/usage/summary       Totals: all time, today, this week — per source
GET  /api/usage/daily         Daily breakdown for charting (last 30 days)
GET  /api/usage/sessions      Paginated session list
GET  /api/usage/projects      Per-project rollup
```

## Dashboard

Single-page UI with:
- Summary cards: tokens today / this week / all time (Claude vs Codex vs pi)
- Bar chart: daily input vs output tokens (last 30 days)
- Table: recent sessions — project, model, source, token counts
- Table: per-project totals

## Token Field Reference

**Claude Code** (`assistant` message entries):
```json
"usage": {
  "input_tokens": 10,
  "cache_creation_input_tokens": 20744,
  "cache_read_input_tokens": 0,
  "output_tokens": 130
}
```

**Codex CLI** (`token_count` event_msg entries):
```json
"total_token_usage": {
  "input_tokens": 20846,
  "cached_input_tokens": 10240,
  "output_tokens": 191,
  "reasoning_output_tokens": 0,
  "total_tokens": 21037
}
```

**pi.dev** (`message` entries where `role` is `assistant`):
```json
"usage": {
  "input": 20846,
  "output": 191,
  "cacheRead": 10240,
  "cacheWrite": 0,
  "totalTokens": 21037,
  "cost": {
    "input": 0.0625,
    "output": 0.573,
    "cacheRead": 0.0075,
    "cacheWrite": 0.0,
    "total": 0.643
  }
}
```

Pi session files are stored under `~/.pi/agent/sessions/--<cwd>--/<timestamp>_<uuid>.jsonl`. Each line is a JSON object with a `type` field. Assistant messages have `type: "message"` and `message.role: "assistant"`. The `usage` object uses camelCase keys (`input`, `output`, `cacheRead`, `cacheWrite`) and includes per-token pricing under `cost`.
